# Permission Retention Verification

목표: 한 번 허용한 권한이 앱 백그라운드 전환, 프로세스 종료, 알림 진입, 앱 재실행 이후에도 유지되는지 확인한다.

## 확인 대상

- 사용정보 접근 권한: Android Settings > Usage access > Screen Time Manager
- 알림 권한: Android 13 이상 `POST_NOTIFICATIONS`
- 앱 알림 허용 상태: Android Settings > Notifications > Screen Time Manager
- 알림 채널 상태: `Screen Time Alerts` 채널이 차단되어 있지 않은지
- WorkManager 정책 체크: 앱이 백그라운드에 있어도 주기 체크가 유지되는지

## 원칙

- 앱은 사용정보 접근 권한이나 알림 권한을 직접 보관하지 않는다. 실제 권한 상태는 Android 시스템이 보관한다.
- 앱 삭제 후 재설치는 새 설치로 간주한다. 이 경우 권한이 유지되지 않는 것이 정상이다.
- 사용자가 권한을 직접 해제하거나, OS가 앱 알림 또는 알림 채널을 차단하면 앱은 이를 권한 없음으로 표시해야 한다.
- 앱 백그라운드 전환, 최근 앱에서 제거, 프로세스 종료만으로 권한이 사라지면 안 된다.

## 사전 준비

1. 앱 설치
2. 사용정보 접근 권한 허용
3. 알림 권한 허용
4. 설정 탭 > 알림 설정에서 `경고 알림`, `초과 알림` ON
5. Safety 탭 > 차단 준비 상태에서 다음 항목 확인
   - 사용정보 권한: OK
   - 알림 권한: OK

## 검증 1: 백그라운드 전환

1. 앱 실행
2. Overview 탭에서 오늘 사용량이 표시되는지 확인
3. Home 버튼으로 앱을 백그라운드 전환
4. 30초 대기
5. 다시 앱 실행
6. Safety 탭에서 사용정보 권한과 알림 권한이 OK인지 확인
7. Overview 탭에서 오늘 사용량이 권한 요청 화면으로 바뀌지 않는지 확인

기대 결과:
- 사용정보 권한 OK 유지
- 알림 권한 OK 유지
- 오늘 사용량이 정상 표시

## 검증 2: 최근 앱에서 제거

1. 앱 실행
2. 최근 앱 화면에서 Screen Time Manager 제거
3. 앱 아이콘으로 다시 실행
4. Safety 탭에서 사용정보 권한과 알림 권한 확인
5. Overview 탭에서 오늘 사용량 새로고침

기대 결과:
- 권한 재요청 없이 사용량 조회 가능
- 알림 권한 OK 유지

## 검증 3: 알림 탭 진입

1. Safe Mode OFF
2. Policy Enforcement ON
3. 경고 또는 초과 조건을 만든다
4. Android 알림이 오면 알림을 터치한다
5. 앱 진입 후 Safety 탭 권한 상태 확인
6. Overview 탭 사용량 확인

기대 결과:
- 알림 터치로 앱 진입
- 사용정보 권한 OK 유지
- 알림 권한 OK 유지
- 사용량 조회 가능

## 검증 4: 앱 알림 또는 채널 차단

1. Android Settings > Notifications > Screen Time Manager 진입
2. 앱 알림 또는 `Screen Time Alerts` 채널을 OFF
3. 앱으로 돌아오기
4. Safety 탭 확인

기대 결과:
- 알림 권한이 WAIT로 표시
- 사용정보 권한은 OK 유지
- 이벤트 로그는 계속 기록 가능
- Android 시스템 알림은 오지 않는 것이 정상

## 검증 5: 사용정보 접근 권한 수동 해제

1. Android Settings > Usage access > Screen Time Manager OFF
2. 앱으로 돌아오기
3. Overview 탭 확인

기대 결과:
- 사용정보 권한 요청 화면 표시
- Safety 탭 사용정보 권한 WAIT
- 알림 권한은 기존 상태 유지

## 실패로 판단할 조건

- 앱을 백그라운드로 보낸 것만으로 사용정보 권한이 WAIT가 됨
- 최근 앱에서 제거 후 재실행했을 뿐인데 알림 권한이 WAIT가 됨
- 알림을 터치해서 앱에 들어온 뒤 오늘 사용량이 장시간 로딩 상태로 멈춤
- 알림 권한은 OK인데 Android 설정에서 앱 알림이나 채널이 꺼져 있음
- 사용정보 권한은 OK인데 Overview에서 권한 설정 버튼이 표시됨

## 추가 확인 메모

- Android가 앱을 강제 중지한 경우 WorkManager 작업은 사용자가 앱을 다시 열 때까지 지연될 수 있다.
- 배터리 최적화, 제조사 백그라운드 제한, 절전 모드는 알림 시점을 늦출 수 있다.
- 하지만 이 경우에도 이미 허용된 사용정보 접근 권한과 알림 권한 자체가 자동으로 해제되면 안 된다.
